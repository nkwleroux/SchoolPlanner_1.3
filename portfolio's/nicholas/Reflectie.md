# Reflectie

In het bedrijsleven wordt gebruik gemaakt van JavaFX. Voor mijn groep hadden we JavaFX voor onze project gebruikt. Met JavaFX konden we vooruit werken vanwege de tutorials online maar in week 10, vlak voor de inleverdatum, kregen we pas van onze senior, Johan, te horen dat het niet mag. We moesten van de school FXGraphics2D gebruiken in plaats van JavaFX. Ik had dus besloten om te onderzoeken waarom de school voorkeur gaf voor FXGraphics2D in plaats van JavaFX.

FXGraphics2D volgens de maker is het een open source "bridge" van de Java 2D API naar JavaFX. Zijn bedrijf wilde de overstap maken van Swing naar JavaFX. Volgens David Gilbert, de schrijver van de artikel die ik had gevonden, was de JavaFX Canvas API anders dan de Java 2D API en biedt geen compatibiliteitslaag. Dus had de bedrijf besloten om FXGraphics2D te maken.

In de artikel werd er geschreven dat veel bedrijven overstapte van Swing naar JavaFX. Ik werd nieuwsgerig over waarom dat zo is. Ik ging dus op zoek naar een artikel die mij veder in de onderwerp kon verdiepen.

JavaFX wordt in de bedrijfsleven gebruikt voor meedere redenen. Het werd gezien als een opvolger van Swing. Vergeleken met Swing biedt JavaFX een duidelijke structuur en bevat het vele verbeteringen zoals styling van componenten, evenementenbeheer, sc√®negrafiek, beter animatie met ingebouwde ondersteuning, en meer. Maar dat is niet te zeggen dat er alleen voordelen zijn voor met het gebruik van JavaFX. In Swing is daar een JOptionPane die wordt gebruikt om korte waarschuwingsberichten weer te geven of om eenvoudige invoer van de gebruiker te krijgen. 

Volgens Oracle is de grootste verbetering van Swing voor JavaFX de browsercomponent (WebView) en de HTML-editorcomponent. De enige belangrijke functie die momenteel ontbreekt, is rich text-ondersteuning voor alle standaardbesturingen, maar dat is aangekondigd voor JavaFX 8.

In conclusie wordt er in het bedrijfsleven gebruik gemaakt van javaFX. In veel gevallen wordt er overstappen gemaakt van bedrijven naar JavaFX van Swing vanwege de verbetere aspecten van JavaFX. In meeste situaties is er voorkeur voor JavaFX maar in sommige gevallen ontbrekt er onderdelen zoals JOptionPane waarbij Swing wordt gebruikt.
